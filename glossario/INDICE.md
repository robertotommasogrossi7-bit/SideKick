# Glossario personale — INDICE

> **Cos'è**: il posto dove finiscono i termini tecnici che Roberto sta imparando, spiegati **semplici**.
> Ogni chat Claude, quando spiega un termine nuovo, lo manda qui. La **raccolta di massa** dai
> materiali di studio (es. AWS) la fa l'osservatorio — vedi `../OSSERVATORIO.md`.
>
> **Come è organizzato**: cartelle per **categoria** (data-engineering, sviluppo-app, java…),
> file per **sotto-tema**. Questo indice dice **dov'è cosa**: tienilo aggiornato ad ogni termine nuovo.

- **`data-engineering/`** — dati, database, pipeline (ingestion → staging → warehouse), AWS.
  - `concetti-fondamentali.md` — OLTP, OLAP, data lake, normalizzazione, star schema, vista materializzata.
  - `aws.md` — *(da popolare dall'osservatorio coi materiali di studio AWS)*.
  - *(previste, si creano quando c'è contenuto: `ingestion.md`, `staging.md`, `database.md`)*
- **`sviluppo-app/`** — costruzione di app e backend.
  - `termini.md` — ORM, layer di sync, UUIDv7, LWW, tombstone, RLS, SSR, EAS/OTA, APK vs AAB, keystore, deep link.
- **`java/`** — *(segnaposto: si riempie quando studi Java)*.
